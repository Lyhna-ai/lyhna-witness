The shape, in three steps

  1. Add Lyhna to your agent's MCP tool path. The proxy sits in the tool-call path your agent already uses — no new platform, no rewrite. It witnesses the real calls and captures the agent's own claims about what it did.
  2. Run one witnessed task. Your agent works as normal. Lyhna records what crossed the tool boundary and what the agent claimed, closes the loop, and emits a witness-input.json.
  3. Read the receipt — then export it. Render the AI Work Receipt with the witness CLI; add --okf --pam to also emit the knowledge and PAM-shaped memory bundles: 
    node src/cli.mjs <witness-input.json> <outDir> --okf --pam
    You get HANDOFF.md, handoff.json, next-ai-prompt.md, plus okf/ and pam/.
