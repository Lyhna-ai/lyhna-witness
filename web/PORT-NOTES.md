# Porting `web/` to the live site (Lyhna Core / lyhna.com)

This `web/` directory is the **canonical reference implementation** of the marketing copy deck. The live
public site lives in a separate repo (**Lyhna Core**, serving `lyhna.com`). When porting this copy/design
onto the live site, preserve the honesty ceiling — it is the product's moat, and it is held *tighter*
than any marketing deck. Do not let "the site already exists" reintroduce overclaims that were
deliberately stripped here.

## What to port

Self-contained, vanilla HTML/CSS/JS — no framework, no build step, relative links, one shared stylesheet:

- `index.html` — homepage (**Lyhna Desktop** framing — "Run your agents. Walk away. Come back to
  receipts."; buy-once ownership + BYO agents/keys/tools; the receipt's 3 verdict states; the honesty
  ceiling as the differentiator; the MCP/OKF/PAM-shaped trio; carrier-vs-witness; demo CTA).
- `demo.html` — the demo, **replayed** (loads `app.js` + `data/handoff.js`).
- `install.html` — honest **connect-your-agent-through-the-MCP-adapter** install (Lyhna Desktop is the
  packaging direction, not a shipped download).
- `pricing.html` — the **ownership model** only (buy once, no numbers).
- `styles.css` — the **receipt grammar**: green/amber/red verdict palette on a clean, monospace-tinged
  document surface. Reuse these tokens; don't drop into a generic SaaS gradient template.
- `app.js`, `data/handoff.js` (generated by `build-data.mjs` from `examples/live-loop/handoff.json`).

## Honesty rules (do NOT relax on the live site)

- **Proof = proof of witnessed tool-boundary action** — never outcome, delivery, correctness, legality,
  or client satisfaction. Prefer **"what crossed the tool boundary"** over "what actually happened."
- Keep **"AI Work Receipt"** as the buyer-facing product object; "proof" is the through-line, not a
  replacement for the receipt.
- **"PAM-shaped" / "portable memory bundle"** — never "PAM-compatible" until validated against a real
  published PAM schema.
- **No pricing numbers** unless intentionally approved for the public site (held here by owner decision —
  the **buy-once ownership model** is published, the numbers are not). Do not reintroduce a metered
  "per-witnessed-action" pricing story or a free tier as the default commercial frame.
- **No fake install promises.** The one-command / "install from lyhna.com" flow and the **Lyhna Desktop
  download** do not exist yet; the install page says **connect through the MCP adapter** and frames the
  desktop app as packaging direction. Keep it honest until the install/download path is real.
- **Say "local by default" / "MCP-compatible", never "local-only".** Cloud-hosted agents may need a
  remote/tunnel bridge later — don't imply every hosted agent connects automatically.
- Demo is a **demo scenario / replay**, not a "rigged" or "planted lie."
- **Kill-list** on every buyer surface: gate · authority · execution authority · governance · judgment
  ledger · IJL · binding · SDK-first · "the layer below human judgment." The proof spine / determinism /
  corpus stay real but *underneath*; a `/how-it-works` deep page may carry the technical truth.

## Overclaims Codex caught here — do not reintroduce on Core

1. **Non-occurrence claim.** The demo must NOT say the email "didn't happen" / "it didn't." Lyhna only
   saw no tool call — use **"Lyhna saw no such call."** (an agent could act through an unwitnessed
   channel; Lyhna can't assert non-occurrence).
2. **Pricing frame.** The commercial story is now **buy once / use with all your agents / unlimited local
   receipts** — a local app you own, not a per-action meter. Do **not** reintroduce metered
   "per-witnessed-action" pricing or a free tier as the default story. (Historical honesty point that still
   holds underneath: a claim with no observed tool call is not a witnessed action — and that absence is
   exactly why the receipt flags it DO-NOT-SEND.)
3. **"Verified memory" tagline.** If the live `lyhna.com` still shows **"Lyhna turns agent work into
   verified memory"** (or any memory-product tagline), **strip it.** That is carrier-lane (Mem0/PAM)
   language; Lyhna is the *witness*, not a memory product. Use proof language — e.g. "here is what was
   actually witnessed." (This phrase is not in this repo; it must be removed on Core.)

## After porting
Run the demo's own dare on the live receipt: copy it, paste it into a fresh AI, ask **"does this
overclaim what Lyhna witnessed?"** — and confirm it comes back clean before shipping.
